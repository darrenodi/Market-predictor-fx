# ModuVise — AI Trading Signals Dashboard

Real-time AI-powered trading signals for BTC, ETH, Gold, and a rotating meme coin. Signals are generated by Claude AI using live price data and news, displayed on a dark-themed dashboard, and pushed to a Telegram group. Built with Next.js 15 and deployed on Vercel.

---

## Features

- **4 asset cards** — BTC/USD, ETH/USD, XAU/USD (Gold), + configurable meme coin slot
- **AI signal generation** — Claude AI analyses prices + news every 30 minutes and outputs: direction (Long/Short), leverage, portfolio %, Take Profit, Stop Loss
- **Live prices** — CoinGecko free API, auto-refreshes every 30 seconds in the browser
- **Sparkline charts** — 24-hour price history per asset
- **Telegram group bot** — posts new signals when confidence is high, alerts when TP or SL is hit
- **TP/SL monitoring** — price check cron runs every 5 minutes

## Setup Guide

### 1. Clone and install

```bash
git clone https://github.com/your-org/moduvise.git
cd moduvise
npm install
```

### 2. Set up Supabase

1. Create a free project at [supabase.com](https://supabase.com)
2. Go to **SQL Editor** and run the full contents of `supabase/schema.sql`
3. Copy your project URL and keys from **Project Settings → API**

### 3. Create Telegram bot and group

1. Open Telegram and message **@BotFather**
2. Send `/newbot` and follow the prompts — copy the **bot token**
3. Create a new Telegram group and add your bot as an admin
4. Add **@userinfobot** to the group, it will print the **group chat ID** (starts with `-100...`)
5. Remove @userinfobot when done

### 4. Get Anthropic API key

Sign up or log in at [console.anthropic.com](https://console.anthropic.com) and create an API key.

### 5. Configure environment variables

Copy the example file and fill in your values:

```bash
cp .env.local.example .env.local
```

| Variable | Description |
| --- | --- |
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase project URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Supabase anon/public key |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase service role key (server-only) |
| `ANTHROPIC_API_KEY` | Claude AI key |
| `TELEGRAM_BOT_TOKEN` | Token from @BotFather |
| `TELEGRAM_GROUP_ID` | Group chat ID (e.g. `-1001234567890`) |
| `CRON_SECRET` | Random secret to authenticate cron calls |
| `ADMIN_SECRET` | Random secret for admin API calls |

Generate secure secrets:

```bash
openssl rand -hex 32
```

### 6. Run locally

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

To trigger a signal update locally (no auth required in development):
```text
GET http://localhost:3000/api/cron/update-signals
```

## Deploying to Vercel

### Deploy

Connect this repo to Vercel via the dashboard or CLI:

```bash
npm i -g vercel
vercel --prod
```

Add all environment variables in **Vercel → Project → Settings → Environment Variables**.

### Set Telegram webhook

Run this once after your first deploy (replace with your actual domain and token):

```
https://api.telegram.org/bot<TELEGRAM_BOT_TOKEN>/setWebhook?url=https://your-domain.com/api/telegram/webhook
```

### Cron jobs

`vercel.json` is configured with:

| Endpoint | Schedule | Purpose |
|---|---|---|
| `/api/cron/update-signals` | Every 30 min | Fetch news + prices → generate signals → notify Telegram |
| `/api/cron/check-prices` | Every 5 min | Check if TP or SL has been hit |

> **Note:** Cron intervals shorter than 24 hours require the **Vercel Pro plan** ($20/month).
>
> **Free alternative:** Use [cron-job.org](https://cron-job.org) (free) to call both endpoints on the same schedule. Add the header `Authorization: Bearer <CRON_SECRET>` to each request.

## Changing the Meme Coin Slot

The 4th asset card is configurable. To switch it, make an authenticated POST request:

```bash
curl -X POST https://your-domain.com/api/admin/meme-coin \
  -H "Authorization: Bearer <ADMIN_SECRET>" \
  -H "Content-Type: application/json" \
  -d '{"symbol": "PEPE"}'
```

Available coins: `DOGE`, `PEPE`, `WIF`, `SHIB`, `BONK`, `FLOKI`, `SOL`, `TRUMP`

To add more, update the `COINS` map in [app/api/admin/meme-coin/route.ts](app/api/admin/meme-coin/route.ts) with the coin's CoinGecko ID.

## Telegram Bot Commands

Users in the group can send:

| Command | Response |
|---|---|
| `/signals` | Current active signals for all 4 assets |
| `/start` | Same as `/signals` |

The bot automatically posts to the group when:
- A new high-confidence signal is generated (confidence ≥ 55%)
- A Take Profit level is hit
- A Stop Loss level is hit

## API Endpoints

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/api/signals` | Latest active signal per asset |
| `GET` | `/api/prices` | Live prices + 24h sparkline history |
| `GET` | `/api/cron/update-signals` | Generate new signals (cron or manual trigger) |
| `GET` | `/api/cron/check-prices` | Check TP/SL against current prices |
| `POST` | `/api/telegram/webhook` | Telegram bot webhook receiver |
| `GET` | `/api/admin/meme-coin` | Get current meme coin |
| `POST` | `/api/admin/meme-coin` | Change the meme coin slot |

Cron endpoints require `Authorization: Bearer <CRON_SECRET>` in production.
Admin endpoint requires `Authorization: Bearer <ADMIN_SECRET>`.

## Database Schema

Three tables in Supabase:

- **`signals`** — AI-generated trading signals with status tracking (`active`, `tp_hit`, `sl_hit`, `expired`)
- **`price_history`** — Price snapshots every 30 min, used for sparkline charts (auto-purge after 7 days)
- **`config`** — Key/value store for settings like the active meme coin

Row-Level Security is enabled — anonymous users have read-only access. All writes use the service role key server-side.
